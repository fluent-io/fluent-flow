Do not create PRs (or branches) on any repo without Victor's explicit approval first.

**Why:** Victor closed PR #4 on fluent-hive that was created without asking. Creating PRs is a visible action that affects shared state.

**How to apply:** When a fix requires a PR, describe what you'd create and ask for confirmation. This applies especially to repos outside the current working directory (e.g. fluent-hive), but also to fluent-flow itself.
